# Agent Naming Guide

Reference for naming future agents on the Engineering Team. Names aren't arbitrary — they shape how people perceive and trust each agent.

## Naming Principles

1. **One word, 1-2 syllables.** Single-syllable names are remembered 20-30% better. Current team averages 1.3 syllables.
2. **Physical/tangible metaphors.** Forge, Spine, Volt — things you can feel. Grounded, not abstract.
3. **No rhyming or pattern overlap.** Each name stands alone in memory. No -er/-or suffixes, no shared endings.
4. **Evokes the domain instantly.** Hearing the name should trigger the right mental image before you read the description.
5. **No trademark collisions.** Avoid names of existing tech products (Sentinel → Microsoft, Nexus → Sonatype, Sentry → sentry.io, Vault → HashiCorp, Pixel → Google, Swift → Apple).

## Phonetic Guidelines

| Sound Type                   | Effect                          | Example             |
| ---------------------------- | ------------------------------- | ------------------- |
| Hard consonants (K, G, T, V) | Forceful, decisive, trustworthy | Forge, Volt, Cortex |
| Soft consonants (L, M, N, S) | Calm, observant, steady         | Vigil, Prism, Atlas |
| Short vowels                 | Snappy, energetic               | Flux, Chip, Tap     |
| Long vowels                  | Expansive, structural           | Spine, Relay        |

## Archetypal Balance

A good team mixes energies. Track what you have:

| Archetype      | Current Agents | Feeling                    |
| -------------- | -------------- | -------------------------- |
| **Builders**   | Forge, Spine   | Constructive, foundational |
| **Movers**     | Relay, Flux    | Dynamic, momentum          |
| **Guardians**  | Warden, Vigil  | Protective, watchful       |
| **Thinkers**   | Cortex, Atlas  | Analytical, knowledgeable  |
| **Interfaces** | Prism, Touch   | Human-facing, tangible     |
| **Raw Power**  | Volt           | Physical, electrical       |

When adding a new agent, check which archetype bucket is underrepresented.

## Emotional Temperature

Keep the team balanced across these spectrums:

- **Cold/Clinical ↔ Warm/Human** — don't stack too many on either side
- **Passive ↔ Active** — the team should feel dynamic, not sleepy
- **Intimidating ↔ Approachable** — a few hard names (Warden, Cortex) balanced by friendly ones (Touch, Atlas)
